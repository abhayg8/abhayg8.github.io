Caracal is a deterministic distributed database written in C++ . It was created as part of a research project by Prof.Asvhin Goel and his graduate students. I worked on with 3 others for my final year design project. Prior to our work, Caracal used static data partitions. When there is imbalanced load, performance is affected. In our project we implemented data migration between nodes to address this gap. In the benchmarking for Caracal, transactions were all generated prior to execution on all nodes. This is not a good approximation of real-world use cases . Therefore, we also implemented dynamic transaction generation that would allow the generation of skewed loads such that data migration could be tested using the TPCC benchmark. We have also implemented the foundations for migration policy so that peformance gains can be realised.

Geographic Information System built in C++ that provides navigational data, made in a team of three for my Software Design Course. This program provides functionality including path finding and information on points of interest. Data was obtained from APIs that use Open Street Map data. We also added further functionality to these APIs. The GUI was developed using Gtk and pathfinding was implemented using the A* algorithm. Back end was added for multiple destination path finding that could allow functionality to extend to a trip planner.

I developed a compiler to generate code for a C style toy programming language using a both existing frameworks and my own code. First, I implemented lexing using re2c and parsing using Bison to generate the abstract syntax tree (AST). I wrote error checking code in C++ that finds syntax and type errors. Finally I used LLVM to implement code generation from the AST.

For my distributed systems course, I developed a Key Value Store system in Java in a team of 3 students. We implemented a client that can send requests (GET, PUT etc.) and a distributed service that processes client requests that are received over a network connection. The servers are organised into a ring using consistent hashing to ensure balanced load. We also implemented data replication so that a single server does not act as a bottle neck. In order to coordinate between servers, and to add/remove them, we implemented an external configuration service that tracks the system state. Data is persisted to disk to allow for recovery if the system restarts or if failure recovery is required.

Vision Checker is website I develop with 2 other for MedHacks 2020 which allows users to do an at home eye test. The front end was built with HTML5, CSS and Javascript. We connected to the Google Cloud Speech-to-Text API using Python, and Flask to connect the front-end and back-end together. Our project won Google COVID-19 Hackathon Fund Award.
